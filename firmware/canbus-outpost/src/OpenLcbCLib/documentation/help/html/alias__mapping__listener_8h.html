Detailed Description

On-demand alias resolution table for consist listener nodes.

Maps listener Node IDs to their CAN aliases so the CAN TX path can transparently resolve dest_alias when forwarding consist commands. Table is sized to LISTENER_ALIAS_TABLE_DEPTH (USER_DEFINED_MAX_LISTENERS_PER_TRAIN * USER_DEFINED_TRAIN_NODE_COUNT).

Entries are registered when a listener is attached and populated when the corresponding AMD frame arrives. AMR clears individual entries; global AME flushes all aliases (node_ids preserved).

◆ ListenerAliasTable_register()

listener_alias_entry_t * ListenerAliasTable_register ( node_id_t node_id)
extern

Registers a listener Node ID in the table with alias = 0.

First-fit allocation into an empty slot (node_id == 0). If the node_id is already registered, returns the existing entry without modification.

Parameters
node_id48-bit OpenLCB Node ID of the listener.
Returns
Pointer to the listener_alias_entry_t, or NULL if full or invalid.

Algorithm:

  1. Validate node_id is non-zero, return NULL if invalid
  2. Scan for existing entry with matching node_id, return it if found
  3. Scan for first empty slot (node_id == 0), store node_id with alias = 0
  4. Return NULL if no empty slot (table full)
* @param node_id  48-bit OpenLCB Node ID of the listener.
*
Returns
Pointer to the listener_alias_entry_t, or NULL if full or invalid.

Stores a resolved alias for a registered listener Node ID.

Called when an AMD frame arrives. If node_id is not in the table, this is a no-op (the node is not one of our listeners).

Parameters

alias12-bit CAN alias from the AMD source.

Algorithm:

  1. Validate alias is in range 0x001-0xFFF, return if invalid

  3. Store alias if found; no-op if node_id is not in the table

◆ ListenerAliasTable_flush_aliases()

void ListenerAliasTable_flush_aliases ( void )
extern

Zeros all alias fields but preserves registered node_ids.

Called when a global AME (empty payload) is received per CanFrameTransferS. The AMD replies triggered by the global AME will re-populate the aliases via ListenerAliasTable_set_alias().

Algorithm:

  1. Iterate all LISTENER_ALIAS_TABLE_DEPTH entries
  2. Set alias = 0 on each; leave node_id untouched

◆ ListenerAliasTable_check_one_verification()

node_id_t ListenerAliasTable_check_one_verification ( uint8_t current_tick)
extern

Probes one listener entry for alias staleness (round-robin).

Rate-limited to once per USER_DEFINED_LISTENER_PROBE_TICK_INTERVAL ticks. At most one entry probed per call. Returns the Node ID whose alias needs verification, or 0 if nothing to do.

State transitions:

  • Resolved + probe interval elapsed -> Verifying (AME sent)
  • Verifying + timeout elapsed -> Stale (alias cleared)
  • Verifying + AMD arrives (via set_alias) -> Resolved
Parameters
current_tickCurrent value of the global 100ms tick counter.
Returns
node_id_t to probe (caller queues targeted AME), or 0.

Called periodically from the CAN main state machine. Rate-limited to once per USER_DEFINED_LISTENER_PROBE_TICK_INTERVAL ticks. Each call advances a static cursor to the next registered entry with a resolved alias and checks:

Algorithm:

  1. If fewer than PROBE_TICK_INTERVAL ticks have elapsed, return 0
  2. Scan up to LISTENER_ALIAS_TABLE_DEPTH entries from the cursor
  3. For each entry with verify_pending set: check timeout, clear alias if expired, continue to next entry
  4. For each resolved entry not pending: check probe interval, set verify_pending and return node_id if due
  5. Return 0 if no entry needs probing this tick
* @param current_tick  Current value of the global 100ms tick counter.
*
Returns
The node_id_t to probe (caller builds and queues targeted AME), or 0 if nothing to do this tick.
